# Learning notes：YOLOv8实时目标检测1

## Overview 前言

之前初步调用了YOLOv8，实现了简单复现，识别出了我提供图片上的部分内容。接下来我进一步尝试了用YOLOv8做实时目标检测，从摄像头获取视频流并显示检测结果。代码虽然简单，但里面有些细节值得记录一下。

## Codes 代码

```python
from ultralytics import YOLO
import cv2

model = YOLO(r"yolov8n.pt")
results = model(
    source = 0,
    stream = True,
)

for result in results:
    plotted = result.plot()
    cv2.imshow("YOLO can see you", plotted)
    if cv2.waitKey(1) & 0xFF == ord("q"):
        break

cv2.destroyAllWindows()
```

## Something after it 一些思考

### Model 关于模型加载
`yolov8n.pt` 是YOLOv8的nano版本，体积小速度快。第一次运行会自动下载，很方便。但它的识别精度较低、置信度较低，可能只有0.7左右，而s、m、l会更细致（但可能掉帧）。我尝试了使用s和m，但是可能跟我当时网络有关，一直跑不出来……我后面会再处理完善。

### stream=True 这个参数
最开始我没加`stream=True`，结果程序卡住不动。查了下文档才明白，不加的话默认处理单张图片，加了之后返回的是生成器，可以一帧一帧处理视频流。这算是视频处理和图片处理的区别了。

### result.plot() 做了什么
`plot()`直接把边界框、类别标签、置信度都画在图像上了，省得自己用OpenCV去画。返回的是RGB格式的numpy数组，可以直接给OpenCV显示。

### How to quit 退出机制
`cv2.waitKey(1) & 0xFF == ord("q")` 这段是OpenCV的标准写法，`waitKey(1)`返回按键的ASCII码，取低8位是为了兼容64位系统。按q退出，挺方便的同时也合理。

## Questions 遇到的问题

1. **摄像头索引问题**：`source=0` 是默认摄像头，但如果笔记本有多个摄像头（比如自带的和外接的），可能需要改成1或2才能找到对的设备。

2. **模型路径**：正如我上面提到的，目前只是用nano模型，置信度精度较低。我后面会完善这个问题。

3. **显示窗口卡死**：一定要加`cv2.waitKey()`，否则窗口会无响应，因为OpenCV需要这个调用来处理GUI事件。
