### Part 1: Image Enhancement | 图像增强

对于商场室内定位系统，图像增强直接影响 YOLOv8 的检测召回率。So it's pretty important...

#### 1.1 Gamma Correction | Gamma 校正

**原理**：人眼对暗部变化更敏感，而摄像头是线性响应。Gamma 校正通过幂律变换重新分配亮度资源。（反正咱们会用就行，这些复杂的算法原理大佬们已经帮咱搞定了。。。）

**项目应用场景**：

- **商场暗区**：走廊尽头、消防通道、部分店铺门口光照不足。gamma < 1 可以提亮暗部，让原本看不清的招牌显现出来。
- **橱窗高光**：店铺橱窗灯光过亮导致过曝。gamma > 1 可以压暗高光，防止 YOLO 漏检。

查表法（**LUT**）实现：

```python
import cv2
import numpy as np

# 这玩意儿每次要show的时候都要写一遍，巨麻烦，直接函数封装得了。
def cv_show(name, img):
    cv2.imshow(name, img)
    cv2.waitKey(0)
    cv2.destroyAllWindows()

img = cv2.imread(r'D:\python-learn\python-YOLOv8\images\test1.jpg')

# Gamma 校正（查表法）
gamma = 0.5  # gamma<1，提亮暗部——适合商场暗区
lut = np.zeros(256, dtype=np.uint8)
for i in range(256):
    lut[i] = int(((i / 255.0) ** gamma) * 255)
gamma_result = cv2.LUT(img, lut)

res = np.hstack((img, gamma_result))
cv_show('Original vs Gamma=0.5 (Low-light Enhancement)', res)
```

**嵌入式知识储备**：查表法在资源受限的硬件上非常高效——256 字节的数组即可完成实时 Gamma 校正，无需做幂运算。（已经迫不及待想试试了但是硬件现在还不咋会，先学着再说吧哈哈）

#### 1.2 White Balance | 白平衡

说实话，我感觉这板块有点难。。写完代码报错了一个世纪，最后才解决问题。

**原理**：白色物体在不同色温光源下会偏色，白平衡将其**校正回中性**。

**项目应用场景**：

- **商场内灯光类型复杂**：暖黄光（2700K）、冷白光（6500K）、混合光同时存在。
- 同一家店铺的 Logo，在**不同区域**拍摄时颜色差异明显。
- 白平衡校正后，YOLO 提取的颜色特征**更稳定**，跨区域识别鲁棒性更强。

##### Method1：最大值法（Max White）

假设画面中最亮的区域是白色。适合商场中庭、天井等光线充足区域。

```python
rgb = cv2.cvtColor(img, cv2.COLOR_BGR2RGB).astype(np.float32)

max_r = np.max(rgb[:, :, 0])
max_g = np.max(rgb[:, :, 1])
max_b = np.max(rgb[:, :, 2])

gain_r = max_g / max_r if max_r != 0 else 1.0
gain_b = max_g / max_b if max_b != 0 else 1.0

rgb[:, :, 0] = np.clip(rgb[:, :, 0] * gain_r, 0, 255)
rgb[:, :, 2] = np.clip(rgb[:, :, 2] * gain_b, 0, 255)

wb_result = cv2.cvtColor(rgb.astype(np.uint8), cv2.COLOR_RGB2BGR)
cv_show('Max White Balance', np.hstack((img, wb_result)))
```

##### Method2：灰度世界法（Gray World）

一开始听到这名字直接没绷住，不过该说不说是挺形象。

假设整张图像的平均颜色是中性灰。适合色彩丰富的商场场景。

```python
avg_r = rgb[:, :, 0].mean()
avg_g = rgb[:, :, 1].mean()
avg_b = rgb[:, :, 2].mean()
avg_all = (avg_r + avg_g + avg_b) / 3

gain_g = avg_all / avg_g
gain_b = avg_all / avg_b

rgb[:, :, 0] = np.clip(rgb[:, :, 0] * gain_r, 0, 255)
rgb[:, :, 1] = np.clip(rgb[:, :, 1] * gain_g, 0, 255)
rgb[:, :, 2] = np.clip(rgb[:, :, 2] * gain_b, 0, 255)
```

#### 1.3 Contrast Stretching | 对比度拉伸

**原理**：将狭窄的灰度范围线性拉伸到 **0~255** 全范围。

**项目应用场景**：

- 远处的小 Logo 对比度低，边界模糊。
- 对比度拉伸后，Logo 的边缘更锐利，YOLO 的特征提取更有效。

```python
min_val = np.min(img)
max_val = np.max(img)
img_float = img.astype(np.float32)
result_stretch = (img_float - min_val) * 255.0 / (max_val - min_val)
result_stretch = np.clip(result_stretch, 0, 255).astype(np.uint8)

# 或使用 OpenCV 内置函数（我就直接用了，I'm a lazy boyyy）
temp = img.copy()
cv2.intensity_transform.autoscaling(img, temp)
img_stretched = temp
```

#### 1.4 商场定位系统预处理管线 Summary actually

建议顺序：**白平衡（统一色彩）→ Gamma 校正（调整亮度）→ 对比度拉伸（增强边缘）**

```python
def mall_localization_preprocess(img):
    """商场室内定位系统的图像预处理管线"""
    # 1. 白平衡——应对复杂灯光
    wb = cv2.xphoto.createSimpleWB()
    img = wb.balanceWhite(img)
    
    # 2. Gamma 校正——应对暗区
    temp = img.copy()
    cv2.intensity_transform.gammaCorrection(img, temp, gamma=1.2)
    img = temp
    
    # 3. 对比度拉伸——增强远处小标识
    temp = img.copy()
    cv2.intensity_transform.autoscaling(img, temp)
    
    return temp
```

#### 2.1 HSV Color Space | HSV 颜色空间

**组成**：

- **H (Hue，色调)**：0~180，决定是什么颜色
- **S (Saturation，饱和度)**：0~255，颜色纯度
- **V (Value，明度)**：0~255，亮度

- 商场定位系统：先用 HSV 快速筛选出"可能是某品牌 Logo"的颜色区域（如星巴克绿、麦当劳红），再送 YOLO 精判。

#### 2.2 YUV Color Space | YUV 颜色空间

**组成**：

- **Y (Luma，亮度)**：黑白图像
- **U (Chroma Blue)**：蓝色色度分量
- **V (Chroma Red)**：红色色度分量

核心思想：**亮度与颜色分离**处理。

这个在**图像ISP**领域应用极其广泛，我得认真学。

**项目应用场景**：

- SuperAgent 智能体处理视频流时，如果直接对 RGB 降噪，颜色会"混在一起"变脏。
- 在 YUV 空间只对 Y 通道降噪，既能去除噪点，又能保留物体原本的颜色——这对场景理解很重要。

```python
yuv = cv2.cvtColor(img, cv2.COLOR_BGR2YUV)

# 只对 Y 通道降噪
yuv[:, :, 0] = cv2.GaussianBlur(yuv[:, :, 0], (5, 5), 1)

result = cv2.cvtColor(yuv, cv2.COLOR_YUV2BGR)
cv_show('Denoise Y only (Color Preserved)', result)
```

**嵌入式知识储备**：**YUV分离处理**是**硬件ISP**的标准架构。未来做嵌入式视觉时，很多摄像头模组直接输出 YUV 格式，理解这个数据流很重要IMPORTANT!!!

### Part 3: Geometric Transformations | 几何变换

#### 3.1 Affine Transformation | 仿射变换

**特点**：保持**平直性**和**平行性**，用于平移、旋转、缩放。

```python
img = cv2.imread(r'D:\python-learn\python-YOLOv8\images\test8.jpg')
h, w = img.shape[:2]

src = np.float32([[50, 50], [w-50, 50], [50, h-50]])
dst = np.float32([[80, 80], [w-20, 80], [80, h-20]])

M = cv2.getAffineTransform(src, dst)
result_aff = cv2.warpAffine(img, M, (w, h))
cv_show('Affine Transform', result_aff)
```

#### 3.2 Perspective Transformation | 透视变换

**特点**：保持**平直性**但不保持**平行性**，可将斜拍物体**校正为正视角**。

**项目应用场景（IMPORTANT!）**：

- 在商场内用手机拍摄店铺招牌时，往往是从地面向上斜拍，Logo 呈现梯形畸变。
- 这种畸变会导致 YOLO 的检测置信度下降 20%~30%。
- 透视校正成"正视图"后，识别效果显著提升。

这段代码还没太理解，求助AI了，以后应用的时候一定搞懂。

```python
# 模拟斜拍的招牌四个角点（我自己用鼠标锚定位置）
src = np.float32([
    [100, 300],   # 左上（远处）
    [500, 280],   # 右上（远处）
    [520, 480],   # 右下（近处）
    [80, 500]     # 左下（近处）
])

# 计算校正后的宽度和高度
width = int(max(np.linalg.norm(src[1] - src[0]), 
                np.linalg.norm(src[2] - src[3])))
height = int(max(np.linalg.norm(src[3] - src[0]), 
                 np.linalg.norm(src[2] - src[1])))

# 目标：正矩形
dst = np.float32([
    [0, 0],
    [width-1, 0],
    [width-1, height-1],
    [0, height-1]
])

M = cv2.getPerspectiveTransform(src, dst)
result_pers = cv2.warpPerspective(img, M, (width, height))
cv_show('Store Sign Correction (Front View)', result_pers)
```

在**智视寻迹**项目中的价值：

用户用手机拍到的招牌往往是斜的，先透视校正，再送 YOLO 识别，可以显著提升跨角度识别的鲁棒性

#### 3.3 Image Registration | 图像配准（基于 ORB 特征）

**目的**：将两张不同角度拍摄的同一场景对齐。

**项目应用场景**：

- 视觉路标匹配：预先采集商场内各店铺 Logo 的"标准照"，存入数据库。
- 用户拍摄时，用 ORB 将当前画面与数据库中的 Logo 进行特征匹配。
- 匹配成功 → 确定用户当前位置（纯视觉定位，不依赖 GPS）。

具体代码每次直接用了，我只需要调整部分参数。

```python
img1 = cv2.imread('logo_template.jpg', 0)   # 数据库中的标准照
img2 = cv2.imread('user_capture.jpg', 0)    # 用户拍摄的画面

# ORB 特征检测（这个很重要，我在下一篇日志里再呈现具体代码）
orb = cv2.ORB_create()
kp1, des1 = orb.detectAndCompute(img1, None)
kp2, des2 = orb.detectAndCompute(img2, None)

# 特征匹配
bf = cv2.BFMatcher(cv2.NORM_HAMMING, crossCheck=True)
matches = bf.match(des1, des2)
matches = sorted(matches, key=lambda x: x.distance)[:30]

# 计算单应性矩阵
src_pts = np.float32([kp1[m.queryIdx].pt for m in matches]).reshape(-1, 1, 2)
dst_pts = np.float32([kp2[m.trainIdx].pt for m in matches]).reshape(-1, 1, 2)
M, mask = cv2.findHomography(src_pts, dst_pts, cv2.RANSAC, 5.0)

# 将用户拍摄的画面与标准照对齐
aligned = cv2.warpPerspective(img2, M, (img1.shape[1], img1.shape[0]))
```

这是**室内定位**的一种重要思路：Logo = 视觉路标，特征匹配 = 位置确认。

## 💡 Some Problems I Met... | 踩到的坑。。。

### Problem 1: Gamma LUT Index Out of Range

- **What's wrong?** 直接用像素值索引 LUT 时，若像素值超过 255 会报错。
- **Reason** 某些预处理后像素值可能变成 float 或超出 0~255。
- **Solution** 确保输入图像为 uint8 类型。

```python
img = img.astype(np.uint8)
```

### Problem 2: White Balance Gain Causes Overflow

- **What's wrong?** 应用增益后 RGB 值超过 255，显示颜色异常。
- **Reason** 增益 >1 时，像素值线性放大后溢出。
- **Solution** 使用 np.clip() 截断。

```python
rgb[:, :, 0] = np.clip(rgb[:, :, 0] * gain_r, 0, 255)
```

### Problem 3: HSV Hue Range Wraps Around

- **What's wrong?** 红色在 HSV 色环两端（0 和 180），单一 inRange 抓不全。
- **Reason** OpenCV 将 0-360 映射为 0-180，红色跨越了 0 点。
- **Solution** 创建两个 mask 然后进行或运算。

```python
mask = cv2.bitwise_or(mask1, mask2)
```

### Problem 4: Perspective Transform Point Order

- **What's wrong?** 校正后的图像出现离谱的扭曲或镜像。
- **Reason** 四个角点的顺序必须是一致的（顺时针或逆时针），否则单应性矩阵会算错。
- **Solution** 严格按照左上→右上→右下→左下的顺序标注。

## 💡 Learning Insights | 学习心得

### Key Takeaways | 关键收获

- 预处理不是"可选"，而是"必须"：商场光照复杂、拍摄角度多变，不做白平衡和透视校正，YOLO 的检测效果会大打折扣。
- HSV 是 YOLO 的好搭档：先用颜色快速筛一遍，再用 YOLO 精判——两阶段策略可以显著减少计算量，适合部署在资源有限的设备上。
- YUV 分离降噪保护颜色：这对智能体处理视频流很重要——降噪是为了画面干净，但颜色信息不能丢。
- 透视变换是招牌识别的标配：斜拍校正后，YOLO 的置信度能提升 20% 以上，这个预处理步骤性价比极高。
- ORB 为纯视觉定位提供了可能：在没有 GPS 的商场内，Logo 就是天然的视觉路标。

def gamma_correction(img, gamma=1.0):
    """Gamma 校正（LUT 法）"""
    lut = np.zeros(256, dtype=np.uint8)
    for i in range(256):
        lut[i] = int(((i / 255.0) ** gamma) * 255)
    return cv2.LUT(img, lut)

def auto_white_balance(img):
    """自动白平衡"""
    wb = cv2.xphoto.createSimpleWB()
    return wb.balanceWhite(img)

def color_region_proposal(img, target_color='red'):
    """
    基于 HSV 的颜色区域提议（YOLO 前的粗筛）
    返回：候选区域的 mask
    """
    hsv = cv2.cvtColor(img, cv2.COLOR_BGR2HSV)
    
    if target_color == 'red':
        mask1 = cv2.inRange(hsv, (0, 50, 50), (10, 255, 255))
        mask2 = cv2.inRange(hsv, (170, 50, 50), (180, 255, 255))
        return cv2.bitwise_or(mask1, mask2)
    elif target_color == 'green':
        return cv2.inRange(hsv, (35, 50, 50), (85, 255, 255))
    elif target_color == 'blue':
        return cv2.inRange(hsv, (100, 50, 50), (130, 255, 255))
    else:
        return np.ones(img.shape[:2], dtype=np.uint8) * 255

def yuv_denoise_only(img, kernel_size=(5,5)):
    """仅对 Y 通道降噪，保护颜色信息"""
    yuv = cv2.cvtColor(img, cv2.COLOR_BGR2YUV)
    yuv[:,:,0] = cv2.GaussianBlur(yuv[:,:,0], kernel_size, 1)
    return cv2.cvtColor(yuv, cv2.COLOR_YUV2BGR)

def perspective_correct_sign(img, src_pts):
    """
    招牌/Logo 透视校正
    src_pts: 斜拍画面中招牌的四个角点（左上、右上、右下、左下）
    返回: 正视角的招牌图像
    """
    width = int(max(np.linalg.norm(src_pts[1]-src_pts[0]), 
                    np.linalg.norm(src_pts[2]-src_pts[3])))
    height = int(max(np.linalg.norm(src_pts[3]-src_pts[0]), 
                     np.linalg.norm(src_pts[2]-src_pts[1])))
    dst_pts = np.float32([[0,0], [width-1,0], [width-1,height-1], [0,height-1]])
    M = cv2.getPerspectiveTransform(src_pts, dst_pts)
    return cv2.warpPerspective(img, M, (width, height))

def mall_localization_pipeline(img, use_color_proposal=True):
    """
    商场定位系统完整预处理管线
    白平衡 → Gamma → 对比度拉伸 → (可选)颜色提议
    """
    # 1. 白平衡——应对复杂灯光
    img = auto_white_balance(img)
    
    # 2. Gamma 校正——应对暗区
    temp = img.copy()
    cv2.intensity_transform.gammaCorrection(img, temp, gamma=1.2)
    img = temp
    
    # 3. 对比度拉伸——增强边缘
    temp = img.copy()
    cv2.intensity_transform.autoscaling(img, temp)
    img = temp
    
    # 4. 可选：颜色区域提议
    if use_color_proposal:
        mask = color_region_proposal(img, 'red')
        img = cv2.bitwise_and(img, img, mask=mask)
    
    return img
